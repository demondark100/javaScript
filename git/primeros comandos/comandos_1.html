primeros comandos git

ls

con esto veremos los archivos que tenemos.

ls -a

con esto veremos los archivos ocultos.

pwd
con esto sabremos en que carpeta o directorio nos encontramos osea nuestra ubicacion exacta.
cd carpeta/directorio
con esto nos moveremos de carpeta en carpeta.

ejemplo:

cd Downloads/algoxd.txt

cd

con esto retrocederemos hasta el inicio de todo

cd ..

con esto retrocederemos solo una carpeta atras.

mkdir nombre

con esto crearemos un nuevo archivo por ejemplo.

mkdir proyecto_1
git init

con esto vamos a inicializar un nuevo repositorio de git.

code .

con esto abriremos el editor en el lusgar que nos encontremos.

git status

con esto veremos el estado de nuestro repositorio.

git add nombre

con esto seleccionaremos un archivo.

variantes.

git add *archivo

ejemplo:
txt , js , py , html , etc.

git add .

con esto seleccionaremos a todos los archivos que esten en ese enlistado.

git commit -m "mensaje"

ejemplo:

git commit -m "arregle un bug XD"

esto nos servira para poner un mensaje a nuestro repositorio.

rm nombre

ejemplo:

rm archivo.txt
git restore --staged nombre de archivo

ejemplo:

git restore --staged algo.txt

con esto quitaremos algun cambio que hagamos en simples palabras es como un ctr-z

mv nombre de archivo nuevo nombre:

ejemplo:

mv archivo.txt archivo_1.txt

con esto cambiaremos el nombre de un archivo.

git mv nombre de archivo , nuevo nombre

con esto lo haremos de forma mas rapida ya que se guardaran los cambios.

.gitignore

con esto estaremos ignorando un archivo esto nos puede servir para que solo nosotros podamos trabajar con ese archivo y que no se suba al repositorio.

.gitignore debemos crearlo en un archivo nuevo y luego dentro de ese archivo poner la carpeta o archivo que queremos ignorar.

git status -s

con esto nos mostrara el estatus de nuestros repositorios pero de forma mas elegante ðŸ˜ŽðŸ˜ŽðŸ˜Ž

git diff

nos esto veremos de forma mas visual los cambios que hicimos en nuestro archivo.

git log

con esto veremos el historial de cambios y ademas el usuario que los hizo.

git log --oneline

con esto veremos lo mismo pero de forma mas elegante.

git branch

con esto veremos en que rama del proyecto estamos trabajando.

git checkout -b nombre de nueva rama

ejemplo:

git checkout -b nueva_rama

con esto crearemos una nueva rama para desarrollar.

git checkout nombre

ejemplo:

git checkout rama_b

con esto nos moveremos a otra rama.

cat

con esto veremos el contenido de nuestro archivo.

git merge nombre

ejemplo:

git merge rama_b

esto nos seevira para traer los cambios de una rama a otra.
posdata debemos estar en la rama que queramos traer los cambios.

git push

con esto estaremos subiendo los cambios que hagamos al github
git push -u origin nombre

ejemplo:

git push -u origin nuevaRama

con esto subiremos una nueva rama al github.
