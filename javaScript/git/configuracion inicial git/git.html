git

consejo si se trabaja desde windows no es recomendable trabajar desde la terminal por defecto es mejor abrir.

"Git Bash"

y si es linux o mac debemos abrir:

cmd terminal

aqui veremos comandos de git para la configuracion inicial.

ver version de git.

git --version
con esto sabremos la version de git

configuracion global.

git config --global

con esto la configuracion que hagamos se aplicara en todos los proyectos que hagamos.

git config --global user.name "nombre completo"

con name pondremos nuestro nombre de usuario.

git config --global user.email correo

con esto configuraremos nuestro correo no es necesario el uso de comillas.

poner nuestro editor de codigo a git.

git config --global core.editor "code --wait"

con esto le estamos asignando un editor de codigo a git.

git config --global -e

con esto abriremos nuestra configuracion en visual studio code debemos cerrar vsc

configuracion de saltos de linea.

git config --global core.autocrlf sistema operativo

esto dependera del sistrma operativo ejemplo.
si es windows debes poner true
si es mac o linux input
con esto le diremos a git como debe tratar los saltos de linea.

ejemplo windows.

git config --global core.autocrlf true

ejemplo linux mac.

git config --global core.autocrlf input
git config --global -h
con esto podremos jugar con la configuracion de git.
