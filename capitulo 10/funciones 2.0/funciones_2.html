funciones 2.0

funcion flecha.

podemos crear funciones sin necesidad del cuerpo "{}" solo cuando sea una sola expresion y solo funciona con las funciones flecha no con funciones clasicas.

error:

const musica = () => {"riptide"}
console.log(musica())

esto nos retornara undefined.

function musica () "riptide";
console.log(musica())

y aqui directamente nos dara error.

correcto:

const musica = () => "riptide";
console.log(musica())

como ya supondran nos retornara "riptide".

los parentesis "()" son obsionales cuando solo es un parametro esto se puede con funciones flecha no con las funciones normales.

const music = nombre =>{
console.log(nombre)
}
music("Rain Over Me")

const music = nombre => "Rain Over Me"
console.log(music())

funcion flecha no recomendada en:

las funciones flecha no son recomendadas como metodos o como constructores de un objeto.

no sirve como metodo.

error:

const objeto = {
musica: "A Sky Full of Stars",
artusta: "Coldplay",
mostrar: ()=>{
console.log(`el artista es ${this.artusta} y la musica es ${this.musica}`)
}
}
objeto.mostrar()

esto nos mostrara "undefined"

correcto:

const objeto = {
musica: "A Sky Full of Stars",
artusta: "Coldplay",
mostrar: function(){
console.log(`el artista es ${this.artusta} y la musica es ${this.musica}`)
}
}
objeto.mostrar()

no es recomendable la funcion flecha como metodo porque a la hora de trabajar con "this" la funcion flecha no reconoce al objeto si no que va a una escala superior.

por ejemplo:

const objeto1 = {
color: "verde",
mostrar: function (){
console.log(`el color es ${this.color}`)
}
}
objeto1.mostrar()

con la "function" a la hora de usar "this" se va hacia el metodo y no hay ningun problema osea que muestra lo que queremos que muestre.

const objeto2 = {
color: "verde",
mostrar: ()=>{
console.log(`el color es ${this.color}`)
}
}
window.color = "azul"
objeto2.mostrar()
//resultado: "el color es azul"

como hemos dicho con la funcion flecha accedemos al objeto que va a una escala mas arriba en este caso va hasta el objeto window.

no sirve como constructor

incorrecto:

const constructor=(musica,artista)=>{
this.musica = musica,
this.artista = artista
}
const musica = new constructor("Mary On A Cross","Ghost")
console.log(musica)

esto directamente nos mandara un error ya que las funciones flecha no se pueden usar como un constructor.

correcto:

function constructo(musica,artista){
this.musica = musica,
this.artista = artista
}
const music = new constructo("Mary On A Cross","Ghost")
console.log(music)

con las funciones clasicas si nos permite crear un constructor para un objeto.
